1. Overview

Hamster Haven is a cozy 3-D game with two big things kids do: they build a hamster world out of tunnels, ramps, wheels and houses, and then they code a robot pet hamster with drag-and-drop blocks to run the world they made. It's a gentle on-ramp from creative building to real computer-science thinking — sequences, loops, conditionals, and variables — with an animal-habitat theme that ties naturally into science and math. It runs in any web browser with nothing to install, and friends can build together in the same room using a 5-letter code.

Everything below describes tools that really exist in the game — nothing invented.

The tools your students will actually use

Scurry & Explore. Kids drive their own hamster with W A S D, Space to jump, and Shift to dash — starting in a cage and escaping into a whole bedroom.
Build Mode (B). Point and click to place parts on a grid: tube, corner, climb tube, ramp, platform, wheel, house, hay pile, fence. Rotate with R, remove with X. This is where the habitat/course gets designed.
Seeds & Stashing. Hamsters collect sunflower seeds (cheeks puff up, up to 10), then stash them in a house to bank them. A "Havenmates" leaderboard tracks who has stored the most.
Coding Mode — "Hammy's Workshop" ({ } Code). A Scratch-style block editor slides in; kids snap blocks together and press Go to watch a robot pet hamster run their program through the world on the right.
The block palette. Categories mirror real coding: Move (walk, turn, hop, go home to nest), Act (grab seed, stash seeds), Sense (seed ahead?, wall ahead?, at nest?), Control (repeat, forever, if / if-else), Operators, Variables, and Events.
Build together. "Create World" makes a room with a 5-letter code; up to 8 players join to build and play in the same space, with friendly emote buttons (1–4).

3. Three Ready-to-Run Lessons

Lesson 1 · Build a Happy Hamster Home 🏠 (Grades K–2, ~20 min)

Objective · Design a habitat that meets an animal's needs, then give Hammy a 3-step program.
Vocabulary: habitat, needs (food · shelter · exercise), sequence, program, grid.
  1. Talk first: "What does a real hamster need to be happy and healthy?" List food, a home, exercise, room to explore.
  2. Press B for Build Mode. Have students place a house (shelter), a hay pile (food), a wheel (exercise), and a couple of tubes (exploring) on the grid.
  3. Escape the cage and scurry the hamster around the new home with W A S D — does it feel like a happy place?
  4. Open { } Code. Together, drag when ⚑ clicked, then walk forward 3, then grab seed. Press Go.
  5. Predict, then check: did Hammy reach the seed? If not, change the number and try again (first debugging!).
Discussion:
  • Which part of your build takes care of which hamster need?
  • What does the word "program" mean now that you've made one?
  • Your first try didn't work perfectly — what did you change to fix it?

Lesson 2 · Teach Hammy to Fetch 🥜 (Grades 2–4, ~30 min)

Objective · Write a multi-step program that collects a seed and returns it to the nest; add a loop.
Vocabulary: algorithm, decompose, loop / repeat, nest, iterate.
  1. In Build Mode, place one house (this becomes the nest) and scatter a short path of tubes/ramps leading away from it.
  2. Decompose the goal out loud: "Fetch a seed" = go to the seed → grab seed → go home to nest → stash seeds.
  3. Build that sequence under when ⚑ clicked using walk forward, turn right, and the Act blocks. Press Go.
  4. When Hammy bumps a wall or misses, use Stop, adjust a turn or a step count, and re-run. Name this the iterate loop of coding.
  5. Now make him greedy: wrap the walk-and-grab steps in a repeat 4 so he fetches again and again. Watch the cheeks fill.
  6. Challenge: add say "Got it!" or a show happy emote when he stashes.
Discussion:
  • How did breaking "fetch" into small steps make it easier to build?
  • What does the repeat block save you from doing by hand?
  • Where did your program go wrong, and how did you find the bug?

Lesson 3 · The Smart Forager 🌻 (Grades 4–5, ~30 min)

Objective · Use conditionals, a variable, and a loop to build a self-driving forager; compare two algorithms with a fair test.
Vocabulary: conditional (if / else), condition, variable, sensor, efficiency, fair test.
  1. Make a variable called score (the ➕ entry in the Variables palette).
  2. Build a self-driving program: when ⚑ clicked → forever [ if on a seed? then grab seed and change score by 1 ; if wall ahead? then turn right else walk forward 1 ].
  3. Press Go and watch Hammy sense his way around the course on his own. This is automation.
  4. Design a second algorithm (for example, always sweep in straight lines and only turn at the edge). Run each on the same course for the same time — a fair test.
  5. Compare the two score values. Which algorithm was more efficient? Why?
  6. Refine the winner: can a small change (a different turn, an extra condition) collect even more?
Discussion:
  • How does the wall ahead? sensor let Hammy make decisions without you?
  • What made your fair test fair — what did you keep the same?
  • If you could add one new sense block, what would it do and why?

4. Conversation Starters

  • "Walk me through your program like you're the hamster — what do you do first, next, last?"
  • "Where did it go wrong the first time? How did you figure out what to fix?"
  • "You used a repeat (or if) block — what would happen if you took it out?"
  • "Which of a real hamster's needs does your build take care of? What's missing?"
  • "If a friend played your world, what's the trickiest part they'd hit?"
  • "What's one thing you'd add next time, and what block would you need?"

5. Capstone Rubric — "Design & Code Your Own Hamster Challenge"

Students build a small course and write a program that guides Hammy through it to collect and stash at least a few seeds, then explain how it works.

Criterion 🌱 Sunflower Sprout (Emerging) 🌿 Busy Builder (Proficient) 🌻 Master Maker (Exemplary)
Habitat & course design Places a few parts; course is mostly decorative. Builds a clear path with a nest that meets several hamster needs. Purposeful course with challenge (turns, climbs) and every need met; considers the player's experience.
Program / algorithm A short sequence of Move/Act blocks that runs. Uses a loop and completes the fetch-and-stash goal. Combines a loop, a conditional with a sensor, and a variable to forage automatically.
Testing & debugging Runs once; leaves errors in. Tests, finds a bug, and fixes it. Iterates repeatedly and compares two approaches to improve the result.
Explaining the thinking Names a block or two. Walks through the program step by step. Explains why each loop/conditional/variable was chosen and what it does.
